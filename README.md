# 室內配線(五股) 甄試刷題

勞動部勞動力發展署北基宜花金馬分署（五股訓練場）**室內配線班入學甄試**的考古題刷題工具。
把 112 / 113 / 114 三年官方甄試試題（每份 40 題、附官方答案）整理成一份標準題庫，
再產出兩種刷題方式：**Anki 牌組**（通勤零碎時間）＋ **刷題 Web App**（含錯題本、間隔重複、模擬考）。

> ⚠️ 這是「入學甄試」的考古題，不是之後的丙級證照學科題庫。
> 甄試每次考 40 題、單選、每題 2.5 分、30 分鐘、答錯不倒扣。考古題重複率高，集中刷這 120 題效益最大。

## 內容

| 檔案 | 說明 |
|---|---|
| `data/questions.json` | 標準題庫（120 題，含分類、答案、詳解），單一事實來源 |
| `anki/室內配線甄試.apkg` | Anki 牌組，匯入手機 App 即可刷 |
| `web/index.html` | 自帶題庫的刷題 App（單一檔案、離線可用） |
| `build.py` | 由題庫產出上述 Anki + Web，改題庫只改這裡再重跑 |

## 一、用 Anki 刷（通勤推薦）

1. 手機裝 **AnkiDroid**（Android，免費）或 **AnkiMobile**（iOS，付費）；電腦裝 **Anki Desktop**（免費）。
2. 把 `anki/室內配線甄試.apkg` 傳到手機（AirDrop / LINE Keep / 雲端）點開匯入，或電腦匯入後同步。
3. 每張卡：先看題目與四個選項 → 心裡選答案 → 翻面看正解與詳解 → 按「Again / Good / Easy」。
   Anki 會自動用間隔重複安排：**答錯的很快再出現，答對的越隔越久**。

## 二、用 Web App 刷（含錯題本）

**電腦：** 直接用瀏覽器打開 `web/index.html` 即可（不需伺服器）。

**手機（推薦丟上網，走路搭車都能刷）：**
- 最簡單：把整個 `web/` 資料夾丟到 **GitHub Pages** / Netlify / Vercel，得到一個網址，手機加到主畫面當 App。
- 或本機起服務：`cd web && python3 -m http.server 8777`，同網段手機開 `http://<電腦IP>:8777`。

功能：
- **智慧排序練習**：Leitner 間隔重複，錯的、沒做過的、🔥高比重的先出。
- **📖 講解模式**：逐選項解析——為什麼正解對、其他三個各自為什麼錯，計算題附算式。
- **🎧 聽力模式**：語音朗讀（Web Speech / zh-TW），自動連播「題目→選項→正解→說明」，
  可調語速、循環、是否唸詳解，通勤免動手用聽的。（螢幕鎖定會中斷朗讀，建議保持螢幕開啟）
- **🔥 高比重分析**：首頁列出跨年度重複考點（幾乎必考）、各工作項目「每卷約幾題」的比重，可一鍵只練高比重題。
- **只練錯題本**：答錯過且尚未精熟的題目，火力集中。
- **模擬考**：隨機 40 題、即時計分，90 分達標提示。
- **依工作項目 / 依年度**練習，每類顯示精熟進度（年度卡標示各年出題重心）。
- 進度存在瀏覽器 `localStorage`，關掉再開還在。

> 每題的逐選項解析放在 `data/rationales_*.json`，由 `build.py` 合併進題庫與 Anki 卡背面。

## 三、改題庫 / 加題

編輯 `build.py` 裡的 `Q` 清單（每題一列：`(考卷, 題號, 分類, 題目, [四選項], 答案1-4, 詳解)`），然後：

```bash
pip3 install genanki      # 只需一次
python3 build.py          # 重新產出 json + web + apkg
```

## 題庫組成

| 場次(site) | 內容 | 題數 | 答案來源 |
|---|---|---|---|
| **五股**（主池） | 112/113/114 甄試 | 120 | 官方 |
| 花蓮 | 112/113/114/115 甄試 | 159 | 112-114 官方；115 原卷無答案→本專案推算(標註) |
| 宜蘭 | 113 甄試（學科+水電知識） | 40 | 手寫官方答案 |

- 花蓮/宜蘭與五股同屬「北基宜花金馬分署」室內配線班，同格式、高度重疊，作為**補充練習**，
  在 App 中以獨立「🗺 其他場次」區塊呈現，**不併入五股的統計、模擬考與錯題本主池**（`site` 欄位區分）。
- 更正：`花蓮112-05`（鉤式電流表一次側匝數）原卷答案疑誤，已依物理更正為 1 匝並註記。
- 其他場次的整理流程在 `build_extra.py`（含自動分類、答案、一致性檢查），逐選項解析在 `data/extra_rationales_*.json`。
